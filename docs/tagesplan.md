# Tagesplan Projektwoche

## Montag

Am Montag wird das Projekt geplant und vorbereitet.

Zuerst wird die Projektidee festgelegt.
Danach wird überlegt, welche Daten und Funktionen die Anwendung braucht.

Außerdem wird die Projektstruktur erstellt und das GitHub Repository vorbereitet.

Aufgaben für Montag:

* Projektidee festlegen
* fachliches Modell definieren
* Projektordner erstellen
* GitHub Repository vorbereiten
* erste Backend-Struktur erstellen

## Dienstag

Am Dienstag liegt der Fokus auf dem Backend.

Es werden die ersten API-Endpunkte erstellt.
Außerdem wird die Speicherung der Aufgaben in einer JSON-Datei umgesetzt.

Danach werden die API-Endpunkte getestet.

Aufgaben für Dienstag:

* REST-API für Aufgaben erstellen
* JSON-Datenhaltung umsetzen
* Aufgaben laden und speichern
* erste API-Tests durchführen

## Mittwoch

Am Mittwoch wird das Frontend aufgebaut.

Dazu werden die HTML-Struktur und das CSS-Grundlayout erstellt.
Das Board bekommt drei Spalten: To Do, In Progress und Done.

Die Aufgaben sollen im Frontend sichtbar angezeigt werden.

Aufgaben für Mittwoch:

* Grundlayout mit HTML und CSS erstellen
* Board mit drei Spalten erstellen
* Aufgaben im Browser anzeigen
* erste einfache Gestaltung umsetzen

## Donnerstag

Am Donnerstag werden Frontend und Backend miteinander verbunden.

Das Frontend soll die Aufgaben über die API vom Backend laden.
Außerdem sollen Aufgaben erstellt, gelöscht und in einen anderen Status verschoben werden können.

Das Design wird ebenfalls verbessert.

Aufgaben für Donnerstag:

* Frontend mit Backend verbinden
* Aufgaben über die API laden
* neue Aufgaben erstellen
* Aufgaben löschen
* Status von Aufgaben ändern
* Styling verbessern

## Freitag

Am Freitag wird das Projekt fertiggestellt.

Die Dokumentation wird überprüft und ergänzt.
Außerdem wird das UML-Diagramm fertig gemacht und die Präsentation vorbereitet.

Zum Schluss wird das Projekt final auf GitHub gespeichert.

Aufgaben für Freitag:

* Dokumentation fertigstellen
* UML-Diagramm ergänzen
* Präsentation vorbereiten
* README prüfen
* Projekt auf GitHub finalisieren
